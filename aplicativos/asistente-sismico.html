§ C / Notas de desarrollo

Cómo lo construí.

El aplicativo nace de una metodología propia que combino con asistencia de IA: primero recopilo fuentes bibliográficas serias (Norma E.030, libros de análisis sísmico), las transcribo a Markdown limpio, defino la lógica de cálculo, y recién después escribo el código.

La idea no es reemplazar ETABS ni el criterio del ingeniero — es eliminar las dos horas de Excel manual que se repiten en cada proyecto. La importación directa de los archivos de salida de ETABS fue el cambio que más tiempo ahorra.

El stack es deliberadamente simple: HTML, CSS y JavaScript puros. Cero dependencias, cero instalación, funciona offline. El usuario abre un archivo y trabaja.
